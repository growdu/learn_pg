# M6: 主备与逻辑复制闭环

## 目标

实现 PostgreSQL 主备复制（物理复制）和逻辑复制的后端执行链，前端展示真实复制状态和拓扑图。

## 1. 架构

### ReplicationProvider 扩展

扩展现有 Provider 接口，增加主备复制方法：

```go
type ReplicationProvider interface {
    Provider
    // Physical replication
    StartReplica(ctx context.Context, spec ReplicationSpec, primaryInfo InstanceInfo) (ReplicaInfo, error)
    StopReplica(ctx context.Context, replicaInfo ReplicaInfo) error
    GetReplicationStatus(ctx context.Context, replicaInfo ReplicaInfo) (ReplicationStatus, error)
}

// ReplicationSpec 定义主备/逻辑复制规格
type ReplicationSpec struct {
    Name         string            // 复制名称
    Type         string            // "physical" | "logical"
    PGVersion    string            // "18", "16"
    PrimaryPort  int               // primary 映射端口
    SecondaryPort int              // standby/subscriber 映射端口
    ProviderID   string            // provider ID
}

// ReplicaInfo 包含复制的运行时信息
type ReplicaInfo struct {
    ProviderID      string         // "docker"
    ComposeProject  string         // docker compose 项目名
    PrimaryInfo     InstanceInfo   // primary 实例信息
    SecondaryInfo   InstanceInfo   // standby/subscriber 实例信息
    ReplicationStatus ReplicationStatus
}

// ReplicationStatus 复制状态
type ReplicationStatus struct {
    PrimaryConnected    bool        // primary 是否可达
    SecondaryConnected   bool        // standby 是否可达
    ReplicationWorking   bool        // 复制是否正常运行
    LAG                  string       // 延迟（如 "10 MB"）
    LastHeartbeat        int64        // 最后心跳时间戳
}
```

### Docker Compose 多容器编排

使用 docker-compose 定义多容器集群：

**物理复制 docker-compose.yml**：
```yaml
version: '3.8'
services:
  primary:
    image: postgres:${PGVERSION}
    environment:
      POSTGRES_PASSWORD: ${PASSWORD}
    ports:
      - "${PRIMARY_PORT}:5432"
    volumes:
      - primary_data:/var/lib/postgresql/data
    command: >
      postgres
      -c wal_level=replica
      -c max_wal_senders=10
      -c hot_standby=on
  standby:
    image: postgres:${PGVERSION}
    environment:
      POSTGRES_PASSWORD: ${PASSWORD}
    ports:
      - "${SECONDARY_PORT}:5432"
    volumes:
      - standby_data:/var/lib/postgresql/data
    command: >
      postgres
      -c primary_conninfo=host=primary port=5432 user=postgres password=${PASSWORD}
      -c hot_standby=on
volumes:
  primary_data:
  standby_data:
```

**逻辑复制 docker-compose.yml**：
```yaml
version: '3.8'
services:
  publisher:
    image: postgres:${PGVERSION}
    environment:
      POSTGRES_PASSWORD: ${PASSWORD}
    ports:
      - "${PRIMARY_PORT}:5432"
    volumes:
      - publisher_data:/var/lib/postgresql/data
    command: >
      postgres
      -c wal_level=logical
      -c max_wal_senders=10
      -c max_replication_slots=10
  subscriber:
    image: postgres:${PGVERSION}
    environment:
      POSTGRES_PASSWORD: ${PASSWORD}
    ports:
      - "${SECONDARY_PORT}:5432"
    volumes:
      - subscriber_data:/var/lib/postgresql/data
    command: >
      postgres
      -c hot_standby=on
volumes:
  publisher_data:
  subscriber_data:
```

### 整体销毁（Teardown）

docker-compose down 统一销毁所有容器和数据卷：

```bash
docker compose -p {project_name} down -v  # -v 删除匿名卷
```

## 2. API 变更

### POST /api/provision/replica

请求：
```json
{
  "projectId": "project-xxx",
  "clusterName": "my-replica-cluster",
  "type": "physical" | "logical",
  "runtime": {
    "type": "docker",
    "pgVersion": "18",
    "primaryPort": 5432,
    "secondaryPort": 5433
  }
}
```

响应：
```json
{
  "success": true,
  "projectId": "xxx",
  "clusterId": "xxx",
  "primaryNodeId": "xxx",
  "secondaryNodeId": "xxx",
  "taskId": "task-xxx"
}
```

### POST /api/clusters/:id/teardown

统一销毁接口，销毁整个集群（所有节点）。

请求：
```json
{
  "cleanupData": true
}
```

响应：
```json
{
  "success": true,
  "cleaned": true
}
```

## 3. Task 生命周期

### 进度步骤

| Progress | Message | 说明 |
|----------|---------|------|
| 10 | "创建 Docker Compose 配置" | 生成 docker-compose.yml |
| 30 | "拉取 Docker 镜像" | docker compose pull |
| 50 | "启动主节点" | docker compose up -d primary |
| 60 | "等待主节点就绪" | pg_isready |
| 70 | "启动从节点" | docker compose up -d standby/subscriber |
| 80 | "建立复制连接" | 创建复制槽/订阅 |
| 90 | "验证复制状态" | 查询 pg_stat_replication/pg_subscription |
| 100 | "集群已就绪" | success |

### 失败处理

失败时设置 `Task.Error` 和 `Task.Result`。


### 集群页拓扑图

物理复制拓扑：
```
[Primary:5432] ←── replication ──→ [Standby:5433]
```

逻辑复制拓扑：
```
[Publisher:5432] ---logical replication---> [Subscriber:5433]
```

## 7. 错误处理

- Docker Compose 不可用 → "Docker Compose 不可用"
- 端口冲突 → 自动尝试下一个端口
- 复制建立失败 → "复制连接建立失败: {detail}"
- 主节点不可用 → "主节点未就绪，无法建立复制"
