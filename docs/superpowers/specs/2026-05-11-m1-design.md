# M1: 收回真相源到后端

最后更新：2026-05-11
状态：已批准

---

## 目标

前端不再持有工作区真数据和敏感凭据，刷新页面后以 backend 为唯一真相源。

- 浏览器 DevTools Application > Local Storage 中无 `pgv_workspace_projects` 和 `pgv_profiles`
- `pgv_provision_task` 仍可保留（任务状态是前端展示用的，不含敏感数据）
- 后端提供稳定的工作区读取与写入边界

---

## 后端改动

### 1. 新建 `internal/api/workspace_crud.go`

资源式 CRUD 接口：

```
POST   /api/projects                          创建项目
GET    /api/projects                           列出项目
GET    /api/projects/{id}                      获取单个项目
PUT    /api/projects/{id}                      更新项目（仅 name）
DELETE /api/projects/{id}                      删除项目

POST   /api/projects/{projectId}/clusters      创建集群
PUT    /api/clusters/{clusterId}               更新集群
DELETE /api/clusters/{clusterId}               删除集群

POST   /api/clusters/{clusterId}/nodes         添加节点
PUT    /api/nodes/{nodeId}                     更新节点（含密码写入，不回显）
DELETE /api/nodes/{nodeId}                    删除节点
```

**密码管理**：
- 创建/更新节点时，前端传递明文密码写入后端存储
- GET 接口返回节点信息，但 `password` 字段返回脱敏字符串 `"********"`
- 前端只保留 UI 临时状态（内存），不持久化

### 2. 改造 `internal/api/workspace_store.go`

- 新增 `GetProject(id)` / `GetCluster(id)` / `GetNode(id)` 增量读取
- `appendCluster` / `appendNode` 内部使用增量写（先读目标，再写回）
- 所有内部方法加 `Locked` 后缀，调用方负责加锁

### 3. 改造 `internal/api/handler.go`

- 删除 `PUT /api/workspace/projects` 全量写入入口（或保留仅供迁移，标记 deprecated）
- 路由注册新的资源式接口

## 前端改动

### 4. `App.tsx`

```diff
- const STORAGE_KEY = 'pgv_workspace_projects'
- function loadProjects(): WorkspaceProject[] = localStorage.getItem(...)
- function saveProjects(projects: WorkspaceProject[]) = localStorage.setItem(...)

+ // 启动时从 backend 加载，删掉所有 localStorage workspace 读写
+ async function loadProjectsFromBackend() // 已有，增强
- saveProjects() 调用 → 改为调用资源式 PUT/POST
```

### 5. `ConnectionManager.tsx`

```diff
- const STORAGE_KEY = 'pgv_profiles'
- function loadProfiles() = localStorage.getItem(...)
- function saveProfiles(profiles) = localStorage.setItem(...)

+ // profiles 只存内存，不持久化到 localStorage
+ // 连接成功后的凭据由 backend 保管，前端不再缓存
```

### 6. `pgStore.ts`

```diff
interface PGConfig {
  host, port, user, database
- password  // 删除，不再写入 localStorage
}
+ activeNodeId: string  // 当前激活的 nodeId
```

### 7. `ClusterHomeView.tsx` 和所有观测组件

- 移除所有直接回传 `node.password` 的代码
- 节点激活改为发送 `POST /api/nodes/:id/activate`（只需 nodeId），后端返回连接状态

## 约束

1. 不改变现有 API 的基本形状（仍使用 `/api/workspace/projects` GET）
2. 新增资源式接口与旧接口共存一段时间，待前端全面迁移后逐步废弃
3. 密码始终不在 GET 响应中返回明文
