# M4: 补齐数据模型

## 目标

在 workspace schema 中补齐以下核心类型：
1. `WorkspaceHost` — 宿主机资源
2. 节点通过 `hostId` 引用宿主机
3. 任务 schema 补全（taskType/result/logs）

## 1. WorkspaceHost 类型

### 后端类型定义

```go
// backend/internal/api/workspace_store.go

type workspaceHost struct {
    ID        string `json:"id"`
    Name      string `json:"name"`
    Host      string `json:"host"`      // IP 或主机名
    Port      int    `json:"port"`      // SSH 端口
    SSHUser   string `json:"sshUser"`   // SSH 用户名
    SSHKey    string `json:"sshKey,omitempty"`   // SSH 私钥路径（可选）
    CreatedAt int64  `json:"createdAt"`
}
```

### 前端类型定义

```typescript
// frontend/src/types/host.ts

export interface WorkspaceHost {
  id: string
  name: string
  host: string
  port: number
  sshUser: string
  sshKey?: string
  createdAt: number
}
```

## 2. workspaceEnvelope 结构

```go
type workspaceEnvelope struct {
    SchemaVersion int                  `json:"schemaVersion"` // 当前为 2
    Projects      []workspaceProject   `json:"projects"`
    Hosts         []workspaceHost       `json:"hosts"` // 新增
}
```

## 3. workspaceNode 增加 HostId 引用

```go
// backend
type workspaceNode struct {
    // ... existing fields ...

}
```

```typescript
// frontend
export interface WorkspaceNode {
    // ... existing fields ...

}
```

## 4. 任务 Schema 补全

### 后端

```go
// backend/internal/api/workspace_store.go

type provisionTask struct {
    TaskID     string `json:"taskId"`
    TaskType   string `json:"taskType"`   // "provision.single" | "provision.physical" | "provision.logical" | "discovery.scan" | "discovery.import"
    Status     string `json:"status"`     // "pending" | "running" | "success" | "failed"
    Progress   int    `json:"progress"`   // 0-100
    Message    string `json:"message,omitempty"`
    Result     string `json:"result,omitempty"`   // 结果摘要
    Logs       string `json:"logs,omitempty"`      // 完整日志
    ProjectID  string `json:"projectId,omitempty"`
    ClusterID  string `json:"clusterId,omitempty"`
    NodeIDs    []string `json:"nodeIds,omitempty"`
    Error      string `json:"error,omitempty"`    // 错误信息
    StartedAt  int64  `json:"startedAt,omitempty"`
    FinishedAt int64  `json:"finishedAt,omitempty"`
}
```

### 前端

```typescript
// frontend/src/types/task.ts

export interface Task {
  taskId: string
  taskType: 'provision.single' | 'provision.physical' | 'provision.logical' | 'discovery.scan' | 'discovery.import'
  status: 'pending' | 'running' | 'success' | 'failed'
  progress: number
  message?: string
  result?: string
  logs?: string
  projectId?: string
  clusterId?: string
  nodeIds?: string[]
  error?: string
  startedAt?: number
  finishedAt?: number
}
```

## 5. API 端点设计

### Host CRUD

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/hosts` | 列出所有主机 |
| POST | `/api/hosts` | 创建主机 |
| GET | `/api/hosts/:id` | 获取单个主机 |
| PUT | `/api/hosts/:id` | 更新主机 |
| DELETE | `/api/hosts/:id` | 删除主机 |

### 任务

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/tasks` | 列出任务（支持 status/taskType 过滤） |
| GET | `/api/tasks/:id` | 获取单个任务 |
| DELETE | `/api/tasks/:id` | 删除任务 |
