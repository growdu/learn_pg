# M5: 单机 provision 真实闭环

## 目标

实现真实拉起 PostgreSQL 单机实例的后端执行链，前端通过独立 Task 面板展示状态，用户手动跳转到目标集群。

## 1. 架构

### Provider 抽象

```
ProvisionService
    ├── DockerProvider  — docker CLI
    └── LocalProvider   — pg_ctl
```

统一的 Start/Stop 接口：

```go
type Provider interface {
    Start(ctx context, spec InstanceSpec) (InstanceInfo, error)
    Stop(ctx context, id string) error
    ID() string
}
```

### InstanceSpec

```go
type InstanceSpec struct {
    Name      string            // 实例名称
    PGVersion string            // postgres:18 等
    Port      int               // 映射端口
    DataDir   string            // 数据目录
    Env       map[string]string // 环境变量
}
```

### InstanceInfo

```go
type InstanceInfo struct {
    ProviderID string            // "docker" | "local"
    ContainerID string           // docker: 容器ID; local: 进程ID
    Host      string
    Port      int               // 实际映射端口
    DataDir   string
}
```

## 2. DockerProvider

### Start

执行：
```bash
docker run -d \
  --name pgv-{name}-{timestamp} \
  -e POSTGRES_PASSWORD={password} \
  -e POSTGRES_USER={user} \
  -p {port}:5432 \
  -v {dataDir}:/var/lib/postgresql/data \
  postgres:{pgVersion}
```

返回：ContainerID、映射端口、数据目录

### Stop

执行：
```bash
docker stop {containerId} && docker rm {containerId}
docker volume rm {volumeName}  # 如果用了 named volume
```

清理数据目录（可选，取决于用户选择）

### 日志获取

```bash
docker logs {containerId}
```

## 3. LocalProvider

### Start

1. 创建数据目录：`/var/lib/pgv/{name}-{timestamp}/`
2. 初始化（如果不存在）：`pg_ctl initdb -D {dataDir}`
3. 启动：`pg_ctl start -D {dataDir} -l {dataDir}/postgres.log`
4. 等待就绪：`pg_isready -h 127.0.0.1 -p {port}`

### Stop

```bash
pg_ctl stop -D {dataDir}
```
清理数据目录

### 前提条件

- PostgreSQL 已安装且 `$PATH` 包含 `pg_ctl`、`postgres`
- 数据目录无权限问题

## 4. Task 生命周期

### Task 字段更新

在 `provisionTask` 中新增 `TaskType = "provision.single"`：

```go
h.setProvisionTask(provisionTask{
    TaskID:    taskID,
    TaskType:  "provision.single",
    Status:    "running",
    Progress:  10,
    Message:   "拉取 Docker 镜像",
    Logs:      "",
    Result:    "",
    ProjectID: req.ProjectID,
    ClusterID: cluster.ID,
    NodeIDs:   []string{node.ID},
    StartedAt: time.Now().UnixMilli(),
})
```

### 进度步骤

| Progress | Message | 说明 |
|----------|---------|------|
| 10 | "拉取 Docker 镜像" | docker pull |
| 30 | "创建容器" | docker run |
| 50 | "等待实例就绪" | pg_isready |
| 70 | "写入工作区" | appendCluster |
| 90 | "验证连接" | tryConnectNode |
| 100 | "单机集群已就绪" | success |

### 失败处理

失败时设置 `Task.Error` 和 `Task.Result`：
- `Result`: 简短摘要（如 "镜像拉取失败"）
- `Error`: 完整错误信息

## 5. API 变更

### POST /api/provision/single

请求：
```json
{
  "projectId": "project-xxx",
  "clusterName": "my-cluster",
  "runtime": {
    "type": "docker" | "local",
    "pgVersion": "18",
    "dataDir": "/var/lib/pgv/xxx"  // local 必填
  }
}
```

响应：
```json
{
  "success": true,
  "projectId": "xxx",
  "clusterId": "xxx",
  "nodeId": "xxx",
  "taskId": "task-xxx"
}
```

### POST /api/clusters/:id/teardown

销毁集群：停止所有容器/进程，清理数据目录，更新 workspace 状态。

请求：
```json
{
  "cleanupData": true
}
```

响应：
```json
{
  "success": true,
  "cleaned": true
}
```

## 6. 前端 Task 面板

### Task 面板位置

在 App.tsx 中， provisionTask 旁边或独立区域。

### Task 面板内容

- 任务列表（最近 N 条）
- 每条任务显示：taskId、状态 badge、progress bar、message、创建时间
- 状态 badge：pending(灰)、running(蓝)、success(绿)、failed(红)
- 成功后：显示"进入集群"按钮
- 失败后：显示"重试"按钮（可选）

### 轮询

每 2 秒轮询 `/api/tasks?limit=10&status=running`，如果有 running 任务，更新面板。

### 跳转

点击"进入集群"按钮：
```typescript
navigate(`/workspace/${projectId}/cluster/${clusterId}`)
```

## 9. 错误处理

- Docker 不可用 → 返回明确错误 "Docker 不可用，请确保 Docker 已安装并运行"
- pg_ctl 不存在 → "PostgreSQL 未安装或不在 PATH 中"
- 端口冲突 → 自动尝试下一个端口（5433, 5434...）
- 权限问题 → "数据目录权限不足"
