# 需求文档（need.md）

## 1. 项目目标
构建一个面向 PostgreSQL 内核学习与观测的可视化平台。当前阶段以 **Web MVP** 为主，围绕“项目 -> 集群 -> 节点”的层级提供可观测能力。

## 2. 当前版本定位
当前目标不是一次性完成跨平台 Tauri 全形态，而是先完成可交付的 Web 版本：
- 支持多项目管理
- 支持多集群（物理复制、逻辑复制）
- 支持多节点配置与下钻观测
- 支持关键内核主题可视化（WAL、CLOG、锁、事务、内存等）

## 3. 核心用户场景
1. 教学演示：创建项目和集群，直观展示复制拓扑与同步状态。
2. 联调排障：定位某个节点，查看 SQL/WAL/CLOG/锁/事务状态。
3. 架构理解：在“整体（集群）-> 局部（节点）”间快速切换。

## 4. 功能需求
### 4.1 工作区层级
- 支持“项目”对象：创建、删除、选择。
- 支持“集群”对象：创建、删除、选择，归属项目。
- 支持“节点”对象：新增、删除、编辑连接参数，归属集群。
- 支持“组件”对象：创建、删除、与集群建立关联。

### 4.2 集群类型
- 物理复制集群：primary/standby 角色模型。
- 逻辑复制集群：publisher/subscriber 角色模型。
- 支持模板创建（最少 1 节点，推荐 2 节点起）。

### 4.3 集群可观测
- 集群主页展示：
  - 集群总览指标（节点总数、在线数、异常数）
  - 复制拓扑图（可交互）
  - 同步状态看板（来自后端 cluster overview）
- 复制链路可视化：支持边与节点点击，查看链路详情。

### 4.4 节点可观测
点击节点后进入节点主页，至少支持：
- SQL 控制台
- WAL 查看
- CLOG 查看
- 锁等待图
- 事务状态
- 内存结构

### 4.5 多集群与关系展示
- 一个项目可包含多个集群、多个组件。
- 组件主页需展示“组件-集群-节点”关系。
- 支持组件到集群的关联编辑与关系图展示。

## 5. 非功能需求
- 可用性：页面逻辑清晰，中文界面优先。
- 可扩展性：后续可接入后端持久化、更多观测指标。
- 一致性：`need/design/apply/README` 文档口径一致。

## 6. 本阶段验收标准
1. 可创建项目、多个集群、多个节点并可编辑。
2. 集群主页可看到复制拓扑与同步状态。
3. 可从集群或组件视角下钻到节点观测页面。
4. 关键页面文案与层级为中文且无乱码。
5. 文档与实际实现一致，不夸大“已完成”范围。
