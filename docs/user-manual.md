# PG Kernel Visualizer 用户手册

## 1. 适用范围

本文档面向当前版本的单用户本地工具。

当前版本已经稳定可用的能力：
- 项目 / 集群 / 节点 / 组件工作区
- 手动添加数据库节点
- 后端连接数据库并进入观测
- 集群总览、复制拓扑、节点专题页

当前版本尚未形成稳定闭环的能力：
- 一键拉起单机 PostgreSQL
- 一键拉起主备复制集群
- 一键拉起逻辑复制集群
- 添加宿主机并自动发现实例

因此，本手册优先介绍当前可用路径，并在对应位置标注规划能力。

## 2. 使用前准备

### 2.1 启动系统

开发模式：

```bash
cd backend
go run ./cmd/server
```

前端开发：

```bash
cd frontend
npm install
npm run dev
```

Docker 模式：

```bash
docker compose -f docker-compose.dev.yml up -d
```

### 2.2 访问入口

- 前端页面：`http://localhost` 或 Vite 开发地址
- 后端健康检查：`http://localhost:3000/health`

## 3. 核心概念

### 3.1 项目

项目是顶层工作区单位，用来组织多个集群和组件。

### 3.2 集群

集群用于表示一个 PostgreSQL 拓扑，可能是：
- 单节点
- 物理复制
- 逻辑复制

### 3.3 节点

节点表示一个 PostgreSQL 实例。当前版本中，节点主要通过手动录入连接参数接入。

### 3.4 组件

组件用于表达与数据库集群相关的外部模块，例如采集器、分析器、存储模块。

## 4. 当前推荐使用流程

### 4.1 创建项目

1. 打开项目主页。
2. 点击“新建项目”。
3. 输入项目名称并保存。

### 4.2 创建集群

当前版本推荐：

1. 在项目下点击“新建集群”。
2. 录入集群名称。
3. 选择集群类型：
   - `physical`
   - `logical`

说明：
- 当前“从模板创建”仍主要用于工作区建模。
- 真正的一键拉起能力仍在实现计划中。

### 4.3 手动添加数据库节点

1. 进入集群主页。
2. 点击“添加节点”。
3. 填写以下信息：
   - 节点名称
   - 主机地址
   - 端口
   - 用户名
   - 密码
   - 数据库名
   - 节点角色（如 `primary`、`standby`）
4. 保存节点。

建议：
- 单机数据库可先标记为 `primary`
- 主备环境中再分别标记 `primary / standby`

### 4.4 激活节点连接

1. 在集群主页节点列表中选择目标节点。
2. 触发“连接”或双击节点进入节点主页。
3. 系统会调用后端建立数据库连接。
4. 连接成功后，可进入节点观测页面。

注意：
- 当前版本由后端维护当前激活连接。
- 切换节点时，系统会重新激活对应节点。

## 5. 观测页面说明

### 5.1 集群主页

主要用于查看：
- 集群内节点数量
- 在线 / 异常节点数
- 复制拓扑
- 同步状态
- 链路详情

### 5.2 节点主页

主要用于进入具体专题页，包括：
- SQL 控制台
- WAL 查看
- CLOG 查看
- 锁等待图
- 事务状态
- Buffer 热图
- 内存结构
- 执行计划树

### 5.3 数据状态说明

页面上可能出现以下状态：
- `实时数据`
- `快照数据`
- `暂无数据`
- `接口异常`

这些状态表示当前页的数据来源和可用性，不代表页面本身故障。

## 6. 当前版本限制

1. 浏览器不直接连接数据库，一切连接均由后端发起。
2. 当前最稳定的接入方式是“手动添加数据库节点”。
3. 一键拉起和宿主机扫描接口目前仍在演进中，不建议当作生产功能依赖。
4. 当前产品定位为单用户本地工具，不面向多人共享服务。

## 7. 常见操作建议

### 7.1 先验证连接再看专题页

如果刚添加节点，建议先确认：
- `/health` 正常
- 节点连接成功
- 集群主页能看到节点在线

再进入 SQL / WAL / 锁图等专题页。

### 7.2 使用复制拓扑时的建议

- 物理复制集群应确保角色标记正确
- 逻辑复制集群应确保 `publisher / subscriber` 角色正确
- 如果拓扑异常，先检查节点连接信息和角色设置

### 7.3 无数据不等于错误

部分专题页在数据库空闲时可能没有明显数据，例如：
- 锁等待图
- 事务状态
- Buffer 热图

这通常表示当前没有可展示的采样结果，不一定是系统异常。
